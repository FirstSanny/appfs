220 class Distribution(Distribution_parse_config_files, _Distribution):
221  """Distribution with support for features, tests, and package data
222 
223  This is an enhanced version of 'distutils.dist.Distribution' that
224  effectively adds the following new optional keyword arguments to 'setup()':
225 
226  'install_requires' -- a string or sequence of strings specifying project
227  versions that the distribution requires when installed, in the format
228  used by 'pkg_resources.require()'. They will be installed
229  automatically when the package is installed. If you wish to use
230  packages that are not available in PyPI, or want to give your users an
231  alternate download location, you can add a 'find_links' option to the
232  '[easy_install]' section of your project's 'setup.cfg' file, and then
233  setuptools will scan the listed web pages for links that satisfy the
234  requirements.
235 
236  'extras_require' -- a dictionary mapping names of optional "extras" to the
237  additional requirement(s) that using those extras incurs. For example,
238  this::
239 
240  extras_require = dict(reST = ["docutils>=0.3", "reSTedit"])
241 
242  indicates that the distribution can optionally provide an extra
243  capability called "reST", but it can only be used if docutils and
244  reSTedit are installed. If the user installs your package using
245  EasyInstall and requests one of your extras, the corresponding
246  additional requirements will be installed if needed.
247 
248  'features' **deprecated** -- a dictionary mapping option names to
249  'setuptools.Feature'
250  objects. Features are a portion of the distribution that can be
251  included or excluded based on user options, inter-feature dependencies,
252  and availability on the current system. Excluded features are omitted
253  from all setup commands, including source and binary distributions, so
254  you can create multiple distributions from the same source tree.
255  Feature names should be valid Python identifiers, except that they may
256  contain the '-' (minus) sign. Features can be included or excluded
257  via the command line options '--with-X' and '--without-X', where 'X' is
258  the name of the feature. Whether a feature is included by default, and
259  whether you are allowed to control this from the command line, is
260  determined by the Feature object. See the 'Feature' class for more
261  information.
262 
263  'test_suite' -- the name of a test suite to run for the 'test' command.
264  If the user runs 'python setup.py test', the package will be installed,
265  and the named test suite will be run. The format is the same as
266  would be used on a 'unittest.py' command line. That is, it is the
267  dotted name of an object to import and call to generate a test suite.
268 
269  'package_data' -- a dictionary mapping package names to lists of filenames
270  or globs to use to find data files contained in the named packages.
271  If the dictionary has filenames or globs listed under '""' (the empty
272  string), those names will be searched for in every package, in addition
273  to any names for the specific package. Data files found using these
274  names/globs will be installed along with the package, in the same
275  location as the package. Note that globs are allowed to reference
276  the contents of non-package subdirectories, as long as you use '/' as
277  a path separator. (Globs are automatically converted to
278  platform-specific paths at runtime.)
279 
280  In addition to these new keywords, this class also has several new methods
281  for manipulating the distribution's contents. For example, the 'include()'
282  and 'exclude()' methods can be thought of as in-place add and subtract
283  commands that add or remove packages, modules, extensions, and so on from
284  the distribution. They are used by the feature subsystem to configure the
285  distribution for the included and excluded features.
286  """
