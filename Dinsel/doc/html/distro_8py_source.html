15 """
16 The ``distro`` package (``distro`` stands for Linux Distribution) provides
17 information about the Linux distribution it runs on, such as a reliable
18 machine-readable distro ID, or version information.
19 
20 It is a renewed alternative implementation for Python's original
21 :py:func:`platform.linux_distribution` function, but it provides much more
22 functionality. An alternative implementation became necessary because Python
23 3.5 deprecated this function, and Python 3.7 is expected to remove it
24 altogether. Its predecessor function :py:func:`platform.dist` was already
25 deprecated since Python 2.6 and is also expected to be removed in Python 3.7.
26 Still, there are many cases in which access to Linux distribution information
27 is needed. See `Python issue 1322 <https://bugs.python.org/issue1322>`_ for
28 more information.
29 """

98  Return information about the current Linux distribution as a tuple
99  ``(id_name, version, codename)`` with items as follows:
100

102  :func:`distro.id`. Otherwise, the result of :func:`distro.name`.

124 def id():
125  """
126  Return the distro ID of the current Linux distribution, as a
127  machine-readable string.
128 
129  For a number of Linux distributions, the returned distro ID value is
130  *reliable*, in the sense that it is documented and that it does not change
131  across releases of the distribution.
132 
133  This package maintains the following reliable distro ID values:
134 
135  ============== =========================================
136  Distro ID Distribution
137  ============== =========================================
138  "ubuntu" Ubuntu
139  "debian" Debian
140  "rhel" RedHat Enterprise Linux
141  "centos" CentOS
142  "fedora" Fedora
143  "sles" SUSE Linux Enterprise Server
144  "opensuse" openSUSE
145  "amazon" Amazon Linux
146  "arch" Arch Linux
147  "cloudlinux" CloudLinux OS
148  "exherbo" Exherbo Linux
149  "gentoo" GenToo Linux
150  "ibm_powerkvm" IBM PowerKVM
151  "kvmibm" KVM for IBM z Systems
152  "linuxmint" Linux Mint
153  "mageia" Mageia
154  "mandriva" Mandriva Linux
155  "parallels" Parallels
156  "pidora" Pidora
157  "raspbian" Raspbian
158  "oracle" Oracle Linux (and Oracle Enterprise Linux)
159  "scientific" Scientific Linux
160  "slackware" Slackware
161  "xenserver" XenServer
162  ============== =========================================
163 
164  If you have a need to get distros for reliable IDs added into this set,
165  or if you find that the :func:`distro.id` function returns a different
166  distro ID for one of the listed distros, please create an issue in the
167  `distro issue tracker`_.
168 
169  **Lookup hierarchy and transformations:**
170 
171  First, the ID is obtained from the following sources, in the specified
172  order. The first available and non-empty value is used:
173 
174  * the value of the "ID" attribute of the os-release file,
175 
176  * the value of the "Distributor ID" attribute returned by the lsb_release
177  command,
178 
179  * the first part of the file name of the distro release file,
180 
181  The so determined ID value then passes the following transformations,
182  before it is returned by this method:
183 
184  * it is translated to lower case,
185 
186  * blanks (which should not be there anyway) are translated to underscores,
187 
188  * a normalization of the ID is performed, based upon
189  `normalization tables`_. The purpose of this normalization is to ensure
190  that the ID is as reliable as possible, even across incompatible changes
191  in the Linux distributions. A common reason for an incompatible change is
192  the addition of an os-release file, or the addition of the lsb_release
193  command, with ID values that differ from what was previously determined
194  from the distro release file name.
195  """
196  return _distro.id()

199 def name(pretty=False):
200  """
201  Return the name of the current Linux distribution, as a human-readable
202  string.
203 
204  If *pretty* is false, the name is returned without version or codename.
205  (e.g. "CentOS Linux")
206 
207  If *pretty* is true, the version and codename are appended.
208  (e.g. "CentOS Linux 7.1.1503 (Core)")
209 
210  **Lookup hierarchy:**
211 
212  The name is obtained from the following sources, in the specified order.
213  The first available and non-empty value is used:
214 
215  * If *pretty* is false:
216 
217  - the value of the "NAME" attribute of the os-release file,
218 
219  - the value of the "Distributor ID" attribute returned by the lsb_release
220  command,
221 
222  - the value of the "<name>" field of the distro release file.
223 
224  * If *pretty* is true:
225 
226  - the value of the "PRETTY_NAME" attribute of the os-release file,
227 
228  - the value of the "Description" attribute returned by the lsb_release
229  command,
230 
231  - the value of the "<name>" field of the distro release file, appended
232  with the value of the pretty version ("<version_id>" and "<codename>"
233  fields) of the distro release file, if available.
234  """
235  return _distro.name(pretty)

238 def version(pretty=False, best=False):
239  """
240  Return the version of the current Linux distribution, as a human-readable
241  string.
242 
243  If *pretty* is false, the version is returned without codename (e.g.
244  "7.0").
245 
246  If *pretty* is true, the codename in parenthesis is appended, if the
247  codename is non-empty (e.g. "7.0 (Maipo)").
248 
249  Some distributions provide version numbers with different precisions in
250  the different sources of distribution information. Examining the different
251  sources in a fixed priority order does not always yield the most precise
252  version (e.g. for Debian 8.2, or CentOS 7.1).
253 
254  The *best* parameter can be used to control the approach for the returned
255  version:
256 
257  If *best* is false, the first non-empty version number in priority order of
258  the examined sources is returned.
259 
260  If *best* is true, the most precise version number out of all examined
261  sources is returned.
262 
263  **Lookup hierarchy:**
264 
265  In all cases, the version number is obtained from the following sources.
266  If *best* is false, this order represents the priority order:
267 
268  * the value of the "VERSION_ID" attribute of the os-release file,
269  * the value of the "Release" attribute returned by the lsb_release
270  command,
271  * the version number parsed from the "<version_id>" field of the first line
272  of the distro release file,
273  * the version number parsed from the "PRETTY_NAME" attribute of the
274  os-release file, if it follows the format of the distro release files.
275  * the version number parsed from the "Description" attribute returned by
276  the lsb_release command, if it follows the format of the distro release
277  files.
278  """
279  return _distro.version(pretty, best)

512 class LinuxDistribution(object):
513  """
514  Provides information about a Linux distribution.
515 
516  This package creates a private module-global instance of this class with
517  default initialization arguments, that is used by the
518  `consolidated accessor functions`_ and `single source accessor functions`_.
519  By using default initialization arguments, that module-global instance
520  returns data about the current Linux distribution (i.e. the distro this
521  package runs on).
522 
523  Normally, it is not necessary to create additional instances of this class.
524  However, in situations where control is needed over the exact data sources
525  that are used, instances of this class can be created with a specific
526  distro release file, or a specific os-release file, or without invoking the
527  lsb_release command.
528  """
529 
530  def __init__(self,
531  include_lsb=True,
532  os_release_file='',
533  distro_release_file=''):
534  """
535  The initialization method of this class gathers information from the
536  available data sources, and stores that in private instance attributes.
537  Subsequent access to the information items uses these private instance
538  attributes, so that the data sources are read only once.
539 
540  Parameters:
541 
542  * ``include_lsb`` (bool): Controls whether the
543  `lsb_release command output`_ is included as a data source.
544 
545  If the lsb_release command is not available in the program execution
546  path, the data source for the lsb_release command will be empty.
547 
548  * ``os_release_file`` (string): The path name of the
549  `os-release file`_ that is to be used as a data source.
550 
551  An empty string (the default) will cause the default path name to
552  be used (see `os-release file`_ for details).
553 
554  If the specified or defaulted os-release file does not exist, the
555  data source for the os-release file will be empty.
556 
557  * ``distro_release_file`` (string): The path name of the
558  `distro release file`_ that is to be used as a data source.
559 
560  An empty string (the default) will cause a default search algorithm
561  to be used (see `distro release file`_ for details).
562 
563  If the specified distro release file does not exist, or if no default
564  distro release file can be found, the data source for the distro
565  release file will be empty.
566 
567  Public instance attributes:
568 
569  * ``os_release_file`` (string): The path name of the
570  `os-release file`_ that is actually used as a data source. The
571  empty string if no distro release file is used as a data source.
572 
573  * ``distro_release_file`` (string): The path name of the
574  `distro release file`_ that is actually used as a data source. The
575  empty string if no distro release file is used as a data source.
576 
577  Raises:
578 
579  * :py:exc:`IOError`: Some I/O issue with an os-release file or distro
580  release file.
581 
582  * :py:exc:`subprocess.CalledProcessError`: The lsb_release command had
583  some issue (other than not being available in the program execution
584  path).
585 
586  * :py:exc:`UnicodeError`: A data source has unexpected characters or
587  uses an unexpected encoding.
588  """
589  self.os_release_file = os_release_file or \
590  os.path.join(_UNIXCONFDIR, _OS_RELEASE_BASENAME)
591  self.distro_release_file = distro_release_file or '' # updated later
592  self._os_release_info = self._get_os_release_info()
593  self._lsb_release_info = self._get_lsb_release_info() \
594  if include_lsb else {}
595  self._distro_release_info = self._get_distro_release_info()
596

627  def id(self):
628  """Return the distro ID of the Linux distribution, as a string.
629 
630  For details, see :func:`distro.id`.
631  """
632  def normalize(distro_id, table):
633  distro_id = distro_id.lower().replace(' ', '_')
634  return table.get(distro_id, distro_id)
635 
636  distro_id = self.os_release_attr('id')
637  if distro_id:
638  return normalize(distro_id, NORMALIZED_OS_ID)
639 
640  distro_id = self.lsb_release_attr('distributor_id')
641  if distro_id:
642  return normalize(distro_id, NORMALIZED_LSB_ID)
643 
644  distro_id = self.distro_release_attr('id')
645  if distro_id:
646  return normalize(distro_id, NORMALIZED_DISTRO_ID)
647 
648  return ''

650  def name(self, pretty=False):
651  """
652  Return the name of the Linux distribution, as a string.
653 
654  For details, see :func:`distro.name`.
655  """
656  name = self.os_release_attr('name') \
657  or self.lsb_release_attr('distributor_id') \
658  or self.distro_release_attr('name')
659  if pretty:
660  name = self.os_release_attr('pretty_name') \
661  or self.lsb_release_attr('description')
662  if not name:
663  name = self.distro_release_attr('name')
664  version = self.version(pretty=True)
665  if version:
666  name = name + ' ' + version
667  return name or ''

669  def version(self, pretty=False, best=False):
670  """
671  Return the version of the Linux distribution, as a string.
672 
673  For details, see :func:`distro.version`.
674  """
675  versions = [
676  self.os_release_attr('version_id'),
677  self.lsb_release_attr('release'),
678  self.distro_release_attr('version_id'),
679  self._parse_distro_release_content(
680  self.os_release_attr('pretty_name')).get('version_id', ''),
681  self._parse_distro_release_content(
682  self.lsb_release_attr('description')).get('version_id', '')
683  ]
684  version = ''
685  if best:
686  # This algorithm uses the last version in priority order that has
687  # the best precision. If the versions are not in conflict, that
688  # does not matter; otherwise, using the last one instead of the
689  # first one might be considered a surprise.
690  for v in versions:
691  if v.count(".") > version.count(".") or version == '':
692  version = v
693  else:
694  for v in versions:
695  if v != '':
696  version = v
697  break
698  if pretty and version and self.codename():
699  version = u'{0} ({1})'.format(version, self.codename())
700  return version

702  def version_parts(self, best=False):
703  """

848  @staticmethod
849  def _parse_os_release_content(lines):
850  """
851  Parse the lines of an os-release file.
852 
853  Parameters:
854 
855  * lines: Iterable through the lines in the os-release file.
856  Each line must be a unicode string or a UTF-8 encoded byte
857  string.
858 
859  Returns:
860  A dictionary containing all information items.
861  """
862  props = {}
863  lexer = shlex.shlex(lines, posix=True)
864  lexer.whitespace_split = True
865 
866  # The shlex module defines its `wordchars` variable using literals,
867  # making it dependent on the encoding of the Python source file.
868  # In Python 2.6 and 2.7, the shlex source file is encoded in
869  # 'iso-8859-1', and the `wordchars` variable is defined as a byte
870  # string. This causes a UnicodeDecodeError to be raised when the
871  # parsed content is a unicode object. The following fix resolves that
872  # (... but it should be fixed in shlex...):
873  if sys.version_info[0] == 2 and isinstance(lexer.wordchars, bytes):
874  lexer.wordchars = lexer.wordchars.decode('iso-8859-1')
875 
876  tokens = list(lexer)
877  for token in tokens:
878  # At this point, all shell-like parsing has been done (i.e.
879  # comments processed, quotes and backslash escape sequences
880  # processed, multi-line values assembled, trailing newlines
881  # stripped, etc.), so the tokens are now either:
882  # * variable assignments: var=value
883  # * commands or their arguments (not allowed in os-release)
884  if '=' in token:
885  k, v = token.split('=', 1)
886  if isinstance(v, bytes):
887  v = v.decode('utf-8')
888  props[k.lower()] = v
889  if k == 'VERSION':
890  # this handles cases in which the codename is in
891  # the `(CODENAME)` (rhel, centos, fedora) format
892  # or in the `, CODENAME` format (Ubuntu).
893  codename = re.search(r'(\(\D+\))|,(\s+)?\D+', v)
894  if codename:
895  codename = codename.group()
896  codename = codename.strip('()')
897  codename = codename.strip(',')
898  codename = codename.strip()
899  # codename appears within paranthese.
900  props['codename'] = codename
901  else:
902  props['codename'] = ''
903  else:
904  # Ignore any tokens that are not variable assignments
905  pass
906  return props
