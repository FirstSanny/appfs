94 def _b_orthonormalize(B, blockVectorV, blockVectorBV=None, retInvR=False):
95  if blockVectorBV is None:
96  if B is not None:
97  blockVectorBV = B(blockVectorV)
98  else:
99  blockVectorBV = blockVectorV # Shared data!!!
100  gramVBV = np.dot(blockVectorV.T, blockVectorBV)
101  gramVBV = cholesky(gramVBV)
102  gramVBV = inv(gramVBV, overwrite_a=True)
103  # gramVBV is now R^{-1}.
104  blockVectorV = np.dot(blockVectorV, gramVBV)
105  if B is not None:
106  blockVectorBV = np.dot(blockVectorBV, gramVBV)
107 
108  if retInvR:
109  return blockVectorV, blockVectorBV, gramVBV
110  else:
111  return blockVectorV, blockVectorBV

114 def lobpcg(A, X, B=None, M=None, Y=None,
115  tol=None, maxiter=20,
116  largest=True, verbosityLevel=0,
117  retLambdaHistory=False, retResidualNormsHistory=False):
118  """Locally Optimal Block Preconditioned Conjugate Gradient Method (LOBPCG)
119 
120  LOBPCG is a preconditioned eigensolver for large symmetric positive
121  definite (SPD) generalized eigenproblems.
122 
123  Parameters
124  ----------
125  A : {sparse matrix, dense matrix, LinearOperator}
126  The symmetric linear operator of the problem, usually a
127  sparse matrix. Often called the "stiffness matrix".
128  X : array_like
129  Initial approximation to the k eigenvectors. If A has
130  shape=(n,n) then X should have shape shape=(n,k).
131  B : {dense matrix, sparse matrix, LinearOperator}, optional
132  the right hand side operator in a generalized eigenproblem.
133  by default, B = Identity
134  often called the "mass matrix"
135  M : {dense matrix, sparse matrix, LinearOperator}, optional
136  preconditioner to A; by default M = Identity
137  M should approximate the inverse of A
138  Y : array_like, optional
139  n-by-sizeY matrix of constraints, sizeY < n
140  The iterations will be performed in the B-orthogonal complement
141  of the column-space of Y. Y must be full rank.
142 
143  Returns
144  -------
145  w : array
146  Array of k eigenvalues
147  v : array
148  An array of k eigenvectors. V has the same shape as X.
149 
150  Other Parameters
151  ----------------
152  tol : scalar, optional
153  Solver tolerance (stopping criterion)
154  by default: tol=n*sqrt(eps)
155  maxiter : integer, optional
156  maximum number of iterations
157  by default: maxiter=min(n,20)
158  largest : bool, optional
159  when True, solve for the largest eigenvalues, otherwise the smallest
160  verbosityLevel : integer, optional
161  controls solver output. default: verbosityLevel = 0.
162  retLambdaHistory : boolean, optional
163  whether to return eigenvalue history
164  retResidualNormsHistory : boolean, optional
165  whether to return history of residual norms
166 
167  Examples
168  --------
169 
170  Solve A x = lambda B x with constraints and preconditioning.
171 
172  >>> from scipy.sparse import spdiags, issparse
173  >>> from scipy.sparse.linalg import lobpcg, LinearOperator
174  >>> n = 100
175  >>> vals = [np.arange(n, dtype=np.float64) + 1]
176  >>> A = spdiags(vals, 0, n, n)
177  >>> A.toarray()
178  array([[ 1., 0., 0., ..., 0., 0., 0.],
179  [ 0., 2., 0., ..., 0., 0., 0.],
180  [ 0., 0., 3., ..., 0., 0., 0.],
181  ...,
182  [ 0., 0., 0., ..., 98., 0., 0.],
183  [ 0., 0., 0., ..., 0., 99., 0.],
184  [ 0., 0., 0., ..., 0., 0., 100.]])
185 
186  Constraints.
187 
188  >>> Y = np.eye(n, 3)
189 
190  Initial guess for eigenvectors, should have linearly independent
191  columns. Column dimension = number of requested eigenvalues.
192 
193  >>> X = np.random.rand(n, 3)
194 
195  Preconditioner -- inverse of A (as an abstract linear operator).
196 
197  >>> invA = spdiags([1./vals[0]], 0, n, n)
198  >>> def precond( x ):
199  ... return invA * x
200  >>> M = LinearOperator(matvec=precond, shape=(n, n), dtype=float)
201 
202  Here, ``invA`` could of course have been used directly as a preconditioner.
203  Let us then solve the problem:
204 
205  >>> eigs, vecs = lobpcg(A, X, Y=Y, M=M, tol=1e-4, maxiter=40, largest=False)
206  >>> eigs
207  array([ 4., 5., 6.])
208 
209  Note that the vectors passed in Y are the eigenvectors of the 3 smallest
210  eigenvalues. The results returned are orthogonal to those.
211 
212  Notes
213  -----
214  If both retLambdaHistory and retResidualNormsHistory are True,
215  the return tuple has the following format
216  (lambda, V, lambda history, residual norms history).
217 
218  In the following ``n`` denotes the matrix size and ``m`` the number
219  of required eigenvalues (smallest or largest).
220 
221  The LOBPCG code internally solves eigenproblems of the size 3``m`` on every
222  iteration by calling the "standard" dense eigensolver, so if ``m`` is not
223  small enough compared to ``n``, it does not make sense to call the LOBPCG
224  code, but rather one should use the "standard" eigensolver,
225  e.g. numpy or scipy function in this case.
226  If one calls the LOBPCG algorithm for 5``m``>``n``,
227  it will most likely break internally, so the code tries to call the standard
228  function instead.
229 
230  It is not that n should be large for the LOBPCG to work, but rather the
231  ratio ``n``/``m`` should be large. It you call the LOBPCG code with ``m``=1
232  and ``n``=10, it should work, though ``n`` is small. The method is intended
233  for extremely large ``n``/``m``, see e.g., reference [28] in
234  http://arxiv.org/abs/0705.2626
235 
236  The convergence speed depends basically on two factors:
237 
238  1. How well relatively separated the seeking eigenvalues are
239  from the rest of the eigenvalues.
240  One can try to vary ``m`` to make this better.
241 
242  2. How well conditioned the problem is. This can be changed by using proper
243  preconditioning. For example, a rod vibration test problem (under tests
244  directory) is ill-conditioned for large ``n``, so convergence will be
245  slow, unless efficient preconditioning is used.
246  For this specific problem, a good simple preconditioner function would
247  be a linear solve for A, which is easy to code since A is tridiagonal.
