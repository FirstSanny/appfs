28 def fmin_cobyla(func, x0, cons, args=(), consargs=None, rhobeg=1.0,
29  rhoend=1e-4, iprint=1, maxfun=1000, disp=None, catol=2e-4):
30  """
31  Minimize a function using the Constrained Optimization BY Linear
32  Approximation (COBYLA) method. This method wraps a FORTRAN
33  implementation of the algorithm.
34 
35  Parameters
36  ----------
37  func : callable
38  Function to minimize. In the form func(x, \\*args).
39  x0 : ndarray
40  Initial guess.
41  cons : sequence
42  Constraint functions; must all be ``>=0`` (a single function
43  if only 1 constraint). Each function takes the parameters `x`
44  as its first argument, and it can return either a single number or
45  an array or list of numbers.
46  args : tuple, optional
47  Extra arguments to pass to function.
48  consargs : tuple, optional
49  Extra arguments to pass to constraint functions (default of None means
50  use same extra arguments as those passed to func).
51  Use ``()`` for no extra arguments.
52  rhobeg : float, optional
53  Reasonable initial changes to the variables.
54  rhoend : float, optional
55  Final accuracy in the optimization (not precisely guaranteed). This
56  is a lower bound on the size of the trust region.
57  iprint : {0, 1, 2, 3}, optional
58  Controls the frequency of output; 0 implies no output. Deprecated.
59  disp : {0, 1, 2, 3}, optional
60  Over-rides the iprint interface. Preferred.
61  maxfun : int, optional
62  Maximum number of function evaluations.
63  catol : float, optional
64  Absolute tolerance for constraint violations.
65 
66  Returns
67  -------
68  x : ndarray
69  The argument that minimises `f`.
70 
71  See also
72  --------
73  minimize: Interface to minimization algorithms for multivariate
74  functions. See the 'COBYLA' `method` in particular.
75 
76  Notes
77  -----
78  This algorithm is based on linear approximations to the objective
79  function and each constraint. We briefly describe the algorithm.
80 
81  Suppose the function is being minimized over k variables. At the
82  jth iteration the algorithm has k+1 points v_1, ..., v_(k+1),
83  an approximate solution x_j, and a radius RHO_j.
84  (i.e. linear plus a constant) approximations to the objective
85  function and constraint functions such that their function values
86  agree with the linear approximation on the k+1 points v_1,.., v_(k+1).
87  This gives a linear program to solve (where the linear approximations
88  of the constraint functions are constrained to be non-negative).
89 
90  However the linear approximations are likely only good
91  approximations near the current simplex, so the linear program is
92  given the further requirement that the solution, which
93  will become x_(j+1), must be within RHO_j from x_j. RHO_j only
94  decreases, never increases. The initial RHO_j is rhobeg and the
95  final RHO_j is rhoend. In this way COBYLA's iterations behave
96  like a trust region algorithm.
97 
98  Additionally, the linear program may be inconsistent, or the
99  approximation may give poor improvement. For details about
100  how these issues are resolved, as well as how the points v_i are
101  updated, refer to the source code or the references below.
102 
103 
104  References
105  ----------
106  Powell M.J.D. (1994), "A direct search optimization method that models
107  the objective and constraint functions by linear interpolation.", in
108  Advances in Optimization and Numerical Analysis, eds. S. Gomez and
109  J-P Hennart, Kluwer Academic (Dordrecht), pp. 51-67
110 
111  Powell M.J.D. (1998), "Direct search algorithms for optimization

118  Examples
119  --------
120  Minimize the objective function f(x,y) = x*y subject
121  to the constraints x**2 + y**2 < 1 and y > 0::
122 
123  >>> def objective(x):
124  ... return x[0]*x[1]
125  ...
126  >>> def constr1(x):
127  ... return 1 - (x[0]**2 + x[1]**2)
128  ...
129  >>> def constr2(x):
130  ... return x[1]
131  ...
132  >>> from scipy.optimize import fmin_cobyla
133  >>> fmin_cobyla(objective, [0.0, 0.1], [constr1, constr2], rhoend=1e-7)
134  array([-0.70710685, 0.70710671])
135 
136  The exact solution is (-sqrt(2)/2, sqrt(2)/2).

204  if not disp:
205  iprint = 0

207  # check constraints
208  if isinstance(constraints, dict):
209  constraints = (constraints, )

211  for ic, con in enumerate(constraints):

213  try:
214  ctype = con['type'].lower()

217  except TypeError:
