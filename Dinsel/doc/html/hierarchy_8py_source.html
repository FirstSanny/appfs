474 def linkage(y, method='single', metric='euclidean'):
475  """
476  Performs hierarchical/agglomerative clustering.
477 
478  The input y may be either a 1d compressed distance matrix
479  or a 2d array of observation vectors.
480 
481  If y is a 1d compressed distance matrix,
482  then y must be a :math:`{n \\choose 2}` sized
483  vector where n is the number of original observations paired
484  in the distance matrix. The behavior of this function is very
485  similar to the MATLAB linkage function.
486 
487  A :math:`(n-1)` by 4 matrix ``Z`` is returned. At the
488  :math:`i`-th iteration, clusters with indices ``Z[i, 0]`` and
489  ``Z[i, 1]`` are combined to form cluster :math:`n + i`. A
490  cluster with an index less than :math:`n` corresponds to one of
491  the :math:`n` original observations. The distance between
492  clusters ``Z[i, 0]`` and ``Z[i, 1]`` is given by ``Z[i, 2]``. The
493  fourth value ``Z[i, 3]`` represents the number of original
494  observations in the newly formed cluster.
495 
496  The following linkage methods are used to compute the distance
497  :math:`d(s, t)` between two clusters :math:`s` and
498  :math:`t`. The algorithm begins with a forest of clusters that
499  have yet to be used in the hierarchy being formed. When two
500  clusters :math:`s` and :math:`t` from this forest are combined
501  into a single cluster :math:`u`, :math:`s` and :math:`t` are
502  removed from the forest, and :math:`u` is added to the
503  forest. When only one cluster remains in the forest, the algorithm
504  stops, and this cluster becomes the root.
505 
506  A distance matrix is maintained at each iteration. The ``d[i,j]``
507  entry corresponds to the distance between cluster :math:`i` and
508  :math:`j` in the original forest.
509 
510  At each iteration, the algorithm must update the distance matrix
511  to reflect the distance of the newly formed cluster u with the
512  remaining clusters in the forest.
513 
514  Suppose there are :math:`|u|` original observations
515  :math:`u[0], \\ldots, u[|u|-1]` in cluster :math:`u` and
516  :math:`|v|` original objects :math:`v[0], \\ldots, v[|v|-1]` in
517  cluster :math:`v`. Recall :math:`s` and :math:`t` are
518  combined to form cluster :math:`u`. Let :math:`v` be any
519  remaining cluster in the forest that is not :math:`u`.
520 
521  The following are methods for calculating the distance between the
522  newly formed cluster :math:`u` and each :math:`v`.
523 
524  * method='single' assigns
525 
526  .. math::
527  d(u,v) = \\min(dist(u[i],v[j]))
528 
529  for all points :math:`i` in cluster :math:`u` and
530  :math:`j` in cluster :math:`v`. This is also known as the
531  Nearest Point Algorithm.
532 
533  * method='complete' assigns
534 
535  .. math::
536  d(u, v) = \\max(dist(u[i],v[j]))
537 
538  for all points :math:`i` in cluster u and :math:`j` in
539  cluster :math:`v`. This is also known by the Farthest Point
540  Algorithm or Voor Hees Algorithm.
541 
542  * method='average' assigns
543 
544  .. math::
545  d(u,v) = \\sum_{ij} \\frac{d(u[i], v[j])}
546  {(|u|*|v|)}
547 
548  for all points :math:`i` and :math:`j` where :math:`|u|`
549  and :math:`|v|` are the cardinalities of clusters :math:`u`
550  and :math:`v`, respectively. This is also called the UPGMA
551  algorithm.
552 
553  * method='weighted' assigns
554 
555  .. math::
556  d(u,v) = (dist(s,v) + dist(t,v))/2
557 
558  where cluster u was formed with cluster s and t and v
559  is a remaining cluster in the forest. (also called WPGMA)
560 
561  * method='centroid' assigns
562 
563  .. math::
564  dist(s,t) = ||c_s-c_t||_2
565 
566  where :math:`c_s` and :math:`c_t` are the centroids of
567  clusters :math:`s` and :math:`t`, respectively. When two
568  clusters :math:`s` and :math:`t` are combined into a new
569  cluster :math:`u`, the new centroid is computed over all the
570  original objects in clusters :math:`s` and :math:`t`. The
571  distance then becomes the Euclidean distance between the
572  centroid of :math:`u` and the centroid of a remaining cluster
573  :math:`v` in the forest. This is also known as the UPGMC
574  algorithm.
575 
576  * method='median' assigns :math:`d(s,t)` like the ``centroid``
577  method. When two clusters :math:`s` and :math:`t` are combined
578  into a new cluster :math:`u`, the average of centroids s and t
579  give the new centroid :math:`u`. This is also known as the
580  WPGMC algorithm.
581 
582  * method='ward' uses the Ward variance minimization algorithm.
583  The new entry :math:`d(u,v)` is computed as follows,
584 
585  .. math::
586 
587  d(u,v) = \\sqrt{\\frac{|v|+|s|}
588  {T}d(v,s)^2
589  + \\frac{|v|+|t|}
590  {T}d(v,t)^2
591  - \\frac{|v|}
592  {T}d(s,t)^2}
593 
594  where :math:`u` is the newly joined cluster consisting of
595  clusters :math:`s` and :math:`t`, :math:`v` is an unused
596  cluster in the forest, :math:`T=|v|+|s|+|t|`, and
597  :math:`|*|` is the cardinality of its argument. This is also
598  known as the incremental algorithm.
599 
600  Warning: When the minimum distance pair in the forest is chosen, there
601  may be two or more pairs with the same minimum distance. This
602  implementation may chose a different minimum than the MATLAB
603  version.
604 
605  Parameters
606  ----------
607  y : ndarray
608  A condensed distance matrix. A condensed distance matrix
609  is a flat array containing the upper triangular of the distance matrix.
610  This is the form that ``pdist`` returns. Alternatively, a collection of
611  :math:`m` observation vectors in :math:`n` dimensions may be passed as an
612  :math:`m` by :math:`n` array. All elements of the condensed distance matrix
613  must be finite, i.e. no NaNs or infs.
614  method : str, optional
615  The linkage algorithm to use. See the ``Linkage Methods`` section below
616  for full descriptions.
617  metric : str or function, optional
618  The distance metric to use in the case that y is a collection of
619  observation vectors; ignored otherwise. See the ``pdist``
620  function for a list of valid distance metrics. A custom distance
621  function can also be used.
622 
623  Returns
624  -------
625  Z : ndarray
626  The hierarchical clustering encoded as a linkage matrix.
627 
628  Notes
629  -----
630  1. For method 'single' an optimized algorithm based on minimum spanning
631  tree is implemented. It has time complexity :math:`O(n^2)`.
632  For methods 'complete', 'average', 'weighted' and 'ward' an algorithm
633  called nearest-neighbors chain is implemented. It also has time
634  complexity :math:`O(n^2)`.
635  For other methods a naive algorithm is implemented with :math:`O(n^3)`
636  time complexity.
637  All algorithms use :math:`O(n^2)` memory.
638  Refer to [1]_ for details about the algorithms.
639  2. Methods 'centroid', 'median' and 'ward' are correctly defined only if
640  Euclidean pairwise metric is used. If `y` is passed as precomputed
641  pairwise distances, then it is a user responsibility to assure that
642  these distances are in fact Euclidean, otherwise the produced result
643  will be incorrect.
644 
645  See Also
646  --------
647  scipy.spatial.distance.pdist : pairwise distance metrics
648 
649  References
650  ----------
651  .. [1] Daniel Mullner, "Modern hierarchical, agglomerative clustering
652  algorithms", :arXiv:`1109.2378v1`.
653  """
654  if method not in _LINKAGE_METHODS:
655  raise ValueError("Invalid method: {0}".format(method))
656 
657  y = _convert_to_double(np.asarray(y, order='c'))
658 
659  if y.ndim == 1:
660  distance.is_valid_y(y, throw=True, name='y')
661  [y] = _copy_arrays_if_base_present([y])
662  elif y.ndim == 2:
663  if method in _EUCLIDEAN_METHODS and metric != 'euclidean':
664  raise ValueError("Method '{0}' requires the distance metric "
665  "to be Euclidean".format(method))
666  if y.shape[0] == y.shape[1] and np.allclose(np.diag(y), 0):
667  if np.all(y >= 0) and np.allclose(y, y.T):
668  _warning('The symmetric non-negative hollow observation '
669  'matrix looks suspiciously like an uncondensed '
670  'distance matrix')
671  y = distance.pdist(y, metric)
672  else:
673  raise ValueError("`y` must be 1 or 2 dimensional.")
674 
675  if not np.all(np.isfinite(y)):
676  raise ValueError("The condensed distance matrix must contain only finite values.")
677 
678  n = int(distance.num_obs_y(y))
679  method_code = _LINKAGE_METHODS[method]
680  if method == 'single':
681  return _hierarchy.mst_single_linkage(y, n)
682  elif method in ['complete', 'average', 'weighted', 'ward']:
683  return _hierarchy.nn_chain(y, n, method_code)
684  else:
685  return _hierarchy.fast_linkage(y, n, method_code)

826  def pre_order(self, func=(lambda x: x.id)):
827  """
828  Performs pre-order traversal without recursive function calls.
829 
830  When a leaf node is first encountered, ``func`` is called with
831  the leaf node as its argument, and its result is appended to
832  the list.
833

835 
836  ids = root.pre_order(lambda x: x.id)
837

839  of the tree as they appear from left to right.
840

842  ----------
843  func : function
844  Applied to each leaf ClusterNode object in the pre-order traversal.
845  Given the ``i``-th leaf node in the pre-order traversal ``n[i]``, the
846  result of ``func(n[i])`` is stored in ``L[i]``. If not provided,
847  the index of the original observation to which the node
848  corresponds is used.
849 
850  Returns
851  -------
852  L : list
853  The pre-order traversal.
854 
855  """
856  # Do a preorder traversal, caching the result. To avoid having to do
857  # recursion, we'll store the previous index we've visited in a vector.
858  n = self.count
859 
860  curNode = [None] * (2 * n)
861  lvisited = set()
862  rvisited = set()
863  curNode[0] = self
864  k = 0
865  preorder = []
866  while k >= 0:
867  nd = curNode[k]
868  ndid = nd.id
869  if nd.is_leaf():
870  preorder.append(func(nd))
871  k = k - 1
872  else:

875  lvisited.add(ndid)
876  k = k + 1
877  elif ndid not in rvisited:
878  curNode[k + 1] = nd.right
879  rvisited.add(ndid)
880  k = k + 1
881  # If we've visited the left and right of this non-leaf
882  # node already, go up in the tree.
883  else:
884  k = k - 1
885 
886  return preorder

1005

1119

1121  disjoint clusters ``s`` and ``t``, respectively and

1134  hierarchical clustering defined by the linkage matrix `Z`

1146  original observations :math:`i` and :math:`j`.

1157  Z = _convert_to_double(Z)

1159  _hierarchy.cophenetic_distances(Z, zz, int(n))

1573  the given linkage matrix.
1574

1576  ----------
1577  Z : ndarray
1578  The hierarchical clustering encoded with the matrix returned
1579  by the `linkage` function.
1580  t : float
1581  The threshold to apply when forming flat clusters.
1582  criterion : str, optional

1584  be any of the following values:
1585

1587  descendants have an inconsistent value less than or equal
1588  to `t` then all its leaf descendants belong to the
1589  same flat cluster. When no non-singleton cluster meets
1590  this criterion, every node is assigned to its own
1591  cluster. (Default)
1592 
1593  ``distance`` : Forms flat clusters so that the original
1594  observations in each flat cluster have no greater a
1595  cophenetic distance than `t`.
1596

1598  the cophenetic distance between any two original
1599  observations in the same flat cluster is no more than
1600  ``r`` and no more than `t` flat clusters are formed.

1608 
1609  MR = maxRstat(Z, R, 3)

1613  non-singleton cluster node ``c`` when ``monocrit[i] <=
1614  r`` for all cluster indices ``i`` below and including
1615  ``c``. ``r`` is minimized such that no more than ``t``

1617  monotonic. For example, to minimize the threshold t on
1618  maximum inconsistency values so that no more than 3 flat

1649 
1650  # Since the C code does not support striding using strides.

1662  [R] = _copy_arrays_if_base_present([R])

1666  elif criterion == 'maxclust':
1667  _hierarchy.cluster_maxclust_dist(Z, T, int(n), int(t))
